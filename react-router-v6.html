React Router

Last updated on August 14, 2023

“React router is a library that provides navigational components for React Developers to create single-page applications (SPAs) with dynamic, client-side routing.” — codecademy.com

How to add react router to your project

  • Start by bootstrapping an app with create-react-app or Vite.
  • cd into your new project directory
  • install react-router-dom using npm

Components

Link, NavLink

A Link is similar to an <a> but prevents default behavior of page refresh. NavLink is similar to a Link, but automatically applies an active class to the link clicked.

Navigate

If you return <Navigate to="/absolute-path" />, the path is returned and rendered.

Route

Define Routes in your App.js file. Use the following snippet as a guide:

import React from "react"; import {RouterProvider, createBrowserRouter, createRoutesFromElements, Route} from "react-router-dom"; 
// import all components you will need to route to 
import "./App.css"; 

const router = createBrowserRouter( 
	createRoutesFromElements( 
		<Route path="/" element={<Root />}> 
			<Route path="/absolute-path-1" element={<ComponentOne />} />{" "} 
			<Route path="/absolute-path-2" element={<ComponentTwo />}> {" "} 
				<Route path=":relative-variable-path" element={<ChildComponent />} />{" "} 
			</Route>{" "} 
		</Route> 
	) 
); 
			
function App() { 
	return ( 
		<> 
			<RouterProvider router={router} />{" "} 
		</> 
	); 
} 
			
export default App;

Note that you can have variables in your path name by using a : before the name of your variable in the path. For example:

<Route path="/articles/:title" element={<Article />} />

or

// Nested routes -- use with <Outlet>
<Route path="/categories" element={<Categories />}>
	<Route path=":name" element={<Category />} />
</Route>

Then you can retrieve these values in the child element (in this case, inside Category.js) with the useParams method. Click here to learn more about useParams from the official docs.

errorElement prop

Provide a nice UX for users by defining an errorElement to be displayed in case an error is thrown.

Hooks

useNavigate

One important tip to remember: don’t call useNavigate within a callback. Use the following snippet as a guide for how to use this hook:

import { useNavigate } from "react-router-dom";
export default function Header() {
	const navigate = useNavigate();

	const handleSubmit = (e) => {
		e.preventDefault();
		// handle any submit tasks
		navigate("/absolute-path");
	}

	return (
		<div>
			<h1>Header</h1>
			<form onSubmit={handleSubmit}>
				<button type="submit">
					Submit
				</button>
			</form>
		</div>
	)
}

useSearchParams and createSearchParams

To use these hooks, you should

  1. Define an object representing the query parameters called searchQueryParams
  2. Create a const called searchQueryString, which should assigned as createSearchParams(searchQueryParams)
  3. Call useNavigate and pass an object containing pathname and search keys, where pathname is a string indicating where to navigate to and search is a string indicating the query string to append to the path

See the following code snippet, which follows the steps outlined above:

import { useNavigate, createSearchParams } from 'react-router-dom';  

const navigate = useNavigate();  
const searchQueryParams = {  
  order: 'ASC'  
}  

const searchQueryString = createSearchParams(searchQueryParams);  

navigate({  
  pathname:'/list',  
  search: `?${searchQueryString}`  
})
